Claude Code come editor video (pipeline a skill)

Gist: trasformare Claude Code in un montatore video: butti il girato grezzo in una cartella-progetto piena di skill, codice e istruzioni markdown, e Claude fa tagli, rimozione silenzi, motion graphics, sottotitoli e musica. È un esempio concreto di claude-skills usate per confezionare un intero workflow di produzione.

L'idea di fondo

Il "sistema" è solo una cartella di file (come ogni progetto Claude Code): skill, script, markdown di istruzioni, workflow, preset. Claude legge la cartella e sa cosa fare. Si può costruire da zero (bastano il tutorial + uno screenshot della struttura dato a Claude per lo scaffolding) — la versione "pronta" venduta in una community è comodità, non necessità.

I 7 stadi della pipeline

  1. Intake — registri una clip grezza e dai il path a Claude.
  2. Rough cut — WhisperX trascrive con timestamp a livello di parola; da lì Claude taglia silenzi, brutte take e filler e decide i tagli. Esempio reale: da 4:10 a 0:47, quasi perfetto al primo colpo. Le correzioni si fanno a voce ("qui hai tagliato troppo vicino alla parola").
  3. Graphics — motore HyperFrames (repo gratis): un grafico per segmento (~22 min la prima passata). È lo stadio che separa "AI slop" da video vero e richiede prompting umano.
  4. Second pass grafica — rifinisci grafico per grafico. Trucco costruito nel progetto: ri-renderizza solo il segmento modificato, non tutto il video → da ~tutto a ~3 min.
  5. Captions (solo short-form) — preset con font Coolvetica, box nero, parole animate in ingresso, sfruttando la trascrizione già fatta.
  6. Background music — dai il path del brano, regoli i decibel (es. -23 dB, "appena percettibile").
  7. Export — copia il finale in Downloads mantenendo il progetto intatto per ri-editare.

Dettagli riusabili

  • Preset di formato: short-form explainer (grafica sopra, faccia sotto, caption al centro), TikTok raw (hook + taglio grezzo + caption), long-form.
  • Prerequisiti: HyperFrames, WhisperX, FFmpeg (~5-10 min di setup).
  • Model-agnostico: funziona con qualsiasi modello (Codex, futuri modelli) → migliora da solo quando escono modelli più forti.
  • Tempo: ~2h di avanti-indietro per un'intro molto curata, contro 4-5h a mano (o giorni per chi non sa editare).
  • Estensione: auto-post a Instagram/TikTok/YouTube Shorts via una "auto poster skill" (stessa logica di master-claude-for-marketing-full-course-2026-07-16 con Blot).

Pattern trasferibile

Due idee valgono oltre il video: (1) skill che confezionano un workflow multi-stadio riproducibile, e (2) il re-render incrementale (rifai solo il pezzo cambiato) — un pattern di efficienza applicabile a qualsiasi pipeline costosa. Collega a ai-content-anti-slop: la qualità sta nel prompting umano sullo stadio graphics, non nel primo output automatico.

Tre stack diversi per lo stesso risultato

Girano più video sullo stesso tema, con catene di tool diverse — utile sapere che non c'è UNA ricetta:

  • Stack A — WhisperX + HyperFrames (how-i-fully-automated-my-video-editing-2026-07-16): quello descritto sopra, tutto dentro Claude Code, trascrizione gratuita e locale.
  • Stack B — Descript + Remotion (how-i-fully-automated-my-video-editing-2026-07-19): i tagli del parlato si fanno a mano-ma-veloci in Descript (editor "tipo Word": cancelli le parole dalla trascrizione e sparisce il video), le grafiche le genera Claude Code con Remotion (libreria che fa video scrivendo codice React); audio con FFmpeg, trascrizione con Whisper di OpenAI. Scelta chiave: tiene un passaggio umano sui tagli per verificare che il parlato sia accurato.
  • Stack C — skill "Video Use" + ElevenLabs Scribe (how-to-fully-automate-video-editing-setup-2026-07-19): guida di setup. Si installa una skill gratuita open-source (Video Use) incollando il link GitHub a Claude ("installamelo") — Claude si tira dietro FFmpeg e registra la skill; trascrizione con ElevenLabs Scribe (a pagamento, chiave API) invece di WhisperX. Modello: Claude Fable 5. Slogan: "no terminal, no Premiere".
⚠️ WARN: sulla trascrizione le fonti divergono — WhisperX (gratis, locale) vs Whisper OpenAI vs ElevenLabs Scribe (a pagamento). Tutte danno timestamp per-parola: non è contraddizione, è scelta costo/comodità. Idem sul motore grafico: HyperFrames vs Remotion vs pacchetti di moduli pronti.
