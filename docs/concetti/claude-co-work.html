Claude Co-work (il "sistema operativo" di Claude)

Gist: Claude Co-work è la modalità dell'app desktop di Claude in cui Claude lavora davvero sui file di una cartella del tuo computer (il workspace), non solo chiacchiera. Attorno a questa cartella ruotano quattro pezzi che, messi insieme, trasformano Claude in un piccolo sistema operativo per il lavoro: il workspace (contesto), le skill (task confezionati), gli MCP/connettori (strumenti esterni) e i live artifact + scheduled task (mini-app e automazioni).

Le tre modalità

  • Chat — come il Claude sul web: gli parli, ti risponde in testo. Nient'altro.
  • Co-work — lavori dentro una cartella a cui dai accesso: Claude legge, scrive e modifica file reali sul tuo computer (documenti, immagini, CSV). È qui che si costruisce un "reparto marketing" o un sistema operativo di business.
  • Code (Claude Code) — la modalità più tecnica. Serve quando co-work non ce la fa: ad esempio caricare immagini generate dà errori di sandbox in co-work, quindi per schedulare post con immagini si passa a Claude Code. Anche i siti SEO si costruiscono in Claude Code (vedi seo-con-claude).

Regola pratica emersa: usa quella con cui sei più comodo — la funzione è quasi sempre la stessa, cambia solo l'interfaccia.

Il workspace: cartella + due file di contesto

Il workspace è una cartella con dentro sottocartelle (content, social, email, leads, assets…) e soprattutto due file che danno il contesto a Claude:

  • CLAUDE.md — chi sei, il tuo brand, la tua voce, come vuoi che Claude si comporti. Va tenuto corto: viene riletto a ogni singolo messaggio, quindi paragrafi su paragrafi rallentano tutto. Dritti al punto.
  • memory.md — i fatti da ricordare nel tempo: offerte, risultati, decisioni, persone chiave, storico. Cresce col tempo dicendo a Claude "ricorda questo". È lo stesso principio di agent-memory.

Questo è il livello "wiki/OS" di second-brain-a-livelli e il cuore di loop-engineering: il contesto sta nei file, non nel prompt di turno.

Live artifact — la mini-app con dati freschi

Un live artifact è una piccola applicazione che vive dentro Claude e pesca dati freschi dai connettori collegati. Esempi visti: un cruscotto che traccia i competitor su YouTube (con "outlier score" e bacheca idee), o un cruscotto ROAS che ripesca i dati dal Facebook Ads MCP a ogni ricarica. Spesso si appoggia a un database (Supabase) per archiviare i dati raccolti, così l'artifact li mostra formattati e interattivi. Puoi costruirne uno per qualsiasi cosa: blog tracker, database prodotti, utenti di un SaaS.

Scheduled task — le automazioni ricorrenti

Uno scheduled task è un compito che Claude esegue a un orario/giorno fisso, in automatico e in background (basta tenere aperta l'app desktop). Esempi: ogni mattina alle 7:00 scrapare i competitor; un digest ROAS ogni giorno alle 9:00 verso Slack; un anomaly detector che gira 4 volte al giorno. È il modo per far lavorare Claude anche quando non ci sei.

Il pattern di fondo

Ricorre in tutti i corsi: Claude è il cervello; gli MCP/connettori sono gli strumenti/mani (vedi mcp-connectors); le skill confezionano i task ripetibili (vedi claude-skills); workspace + memory danno il contesto stabile. Tutto il resto è comporre questi quattro mattoni.
